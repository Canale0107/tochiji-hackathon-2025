位置情報に紐づく"リアルタイムな体験と感情"を集約・可視化することで、 人々の日常に発見と共感を生み出し、市民と行政の間に存在する"見えない壁"を溶かす。

なぜこのアプローチが"勝つ"のか

👥

若者の"無関心"の正体

彼らは政治に無関心なのではない。「自分たちの声が届かない」という無力感と、「面白くない」という体験の欠如が原因。

まず「面白い」体験を提供し、その先に自然な形で社会参加があるべき
🏛️

行政DXの罠

多くの行政アプリは「課題解決」を前面に出しすぎるため、利用者が限定される。

「エンタメ・ライフスタイル」の皮を被ったシビックテックとして、あらゆる層を自然に巻き込む
🌆

渋谷のダイナミズム

渋谷の魅力は、日々生まれては消えるポップアップストア、展示、ストリートイベントにある。

この「刹那的な情報」を捉えきれるプラットフォームは存在しない。巨大なチャンス！
🌍

グローバル都市の宿命

外国人住民・観光客の増加は不可逆。彼らの体験や不満は、言語の壁によって可視化されていない。

多言語対応は「付加機能」ではなく、渋谷の未来を語る上での「必須条件」

⚡ コア機能詳細

機能1: 地図タイムライン (The Live Map Timeline)

UI/UX設計

  • 地図がメインのタイムライン表示
  • 時間軸スライダーで過去・現在・未来を操作
  • アニメーションでピンが出現/消失
  • 直感的で楽しいインターフェース

情報レイヤー

  • 公式イベントピン：行政・店舗の登録情報
  • Live Post：ユーザーの「いま、ここ」投稿
  • Quest：お題エリアの可視化
  • ヒートマップ：盛り上がりの可視化

鮮度管理アルゴリズム

  • 時間経過で透明度が減少（フェードアウト）
  • 「いま」の情報が最も目立つ設計
  • 24時間で完全に消失
  • 重要度に応じた減衰速度調整

機能2: エンゲージメントエンジン

機能名 説明 トリガー条件 期待効果
スマート通知 位置・時間・興味に基づく通知 500m以内にイベント発生 参加率30%向上
Quest（お題） ゲーム感覚の市民参加 エリア進入時 投稿数5倍増
バッジシステム 投稿・参加の可視化 特定アクション達成 継続率40%向上
ランキング エリア別貢献度表示 週次/月次更新 競争心による活性化

機能3: ボーダーレスコミュニケーション

多言語対応は必須条件： 渋谷の外国人比率は年々増加。言語の壁を技術で解決することが、真のダイバーシティを実現する。

自動UI切り替え

端末の言語設定を検出し、自動でUI言語を切り替え（日・英・中・韓）

リアルタイム翻訳

DeepL APIによる高精度翻訳。ワンタップで投稿を母国語に変換

文化的配慮

各言語圏の文化に配慮したコンテンツ表示とモデレーション

🎤 ハッカソンプレゼン戦略

勝利への道筋

コンセプトの転換

「行政課題を解決するアプリ」から「渋谷の"いま"を誰もが楽しめる、次世代マップ型SNS」へ

行政課題の解決は、その結果として自然に達成される副産物として語る。

冒頭（30秒）- 問題提起

「皆さん、Googleマップで見る渋谷は、過去の情報の集積です。 Xで見る渋谷は、位置情報がバラバラです。 では、渋谷の"いま"は、どこにあるのでしょうか？ ...どこにもありません。だから、私たちが創ります。」

中盤（3分）- デモ

  • ポジティブでワクワクするデモ
  • リアルタイムでピンが現れる様子
  • 多言語翻訳の驚き体験
  • Quest参加の楽しさ

キーワード：「これは単なる地図アプリではありません。街を舞台にした、リアルタイム・ソーシャルゲームです」

締め（30秒）- ビジョン

「我々が創る『Shibuya Live Canvas』は、若者も、外国人も、お年寄りも、 誰もが主役になれる、渋谷という街の"劇場"です。 行政課題の解決は、この劇場が盛り上がった先に待っている、最高のアンコールに過ぎません。 さあ、一緒に、この舞台の幕を開けましょう。」
差別化ポイント： 技術的な説明は最小限に。体験価値と未来のワクワクを最大限に伝える。 審査員に「これは使ってみたい！」と思わせることが全て。

⚙️ 環境設定

Claude Code統合設定

重要：以下の設定をClaude Codeの設定ファイルまたは環境変数として登録してください
設定項目 値 用途
API_BASE_URL https://api.shibuya-live-canvas.jp APIエンドポイント
DEEPL_API_KEY ****-****-****-**** 翻訳API認証
MAPBOX_TOKEN pk.****-****-**** 地図表示
FIREBASE_CONFIG {...} 認証設定
DATABASE_URL postgresql://... DB接続
REDIS_URL redis://... キャッシュ
AWS_ACCESS_KEY AKIA**** S3アクセス
NODE_ENV development 実行環境

設定コマンド例

# .env.local ファイルに記載
API_BASE_URL=https://api.shibuya-live-canvas.jp
DEEPL_API_KEY=your-deepl-api-key
MAPBOX_TOKEN=your-mapbox-token

# または環境変数として設定
export API_BASE_URL=https://api.shibuya-live-canvas.jp
export DEEPL_API_KEY=your-deepl-api-key
export MAPBOX_TOKEN=your-mapbox-token

開発環境セットアップ

1. リポジトリクローン

git clone https://github.com/shibuya-live-canvas/app.git

2. 依存関係インストール

npm install && cd ios && pod install

3. 環境変数設定

cp .env.example .env.local

4. 開発サーバー起動

npm run dev
