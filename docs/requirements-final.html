💡 プロダクトコンセプト

🎨 The Living Canvas & Real-time SNS

我々が創るのは「Shibuya Live Canvas」。これは、渋谷の"いま"を映し出す「生きたキャンバス」であり、 Googleマップが過去の口コミなら、我々は「現在進行形の体験」を共有する、次世代のマップ型SNS。 ネガティブな課題報告ではなく、ポジティブな発見や感動が主役のプラットフォーム。

🔧 プロダクトの4つの柱

🗺️

マップ型SNS

革新性：位置情報を軸にした新しいソーシャル体験

  • 地図がメインのタイムライン表示
  • 位置に紐づく感情・体験の共有
  • 空間的なコミュニケーション

リアルタイム性

差別化：「いま」起きていることの即座な共有

  • フェードアウト機能で鮮度管理
  • 時間軸スライダーで過去・未来表示
  • ライブ感のある情報更新

ゲーミフィケーション

エンゲージメント：Quest機能で楽しく市民参加

  • 期間限定チャレンジ
  • バッジ・ランキングシステム

🎬 ユースケースシナリオ

📅 A Day in the Life - 24時間のユーザーストーリー

異なるペルソナが同じプラットフォーム上でどのような価値を見出し、どう相互作用するかを時系列で追う。

🌅 8:00 - 通学路の発見（ユウキ・大学生）

シチュエーション：いつもの通学路でアプリを起動

発見：「駅前に新しいタピオカスタンドがオープン！今日限定50%OFF」の投稿を発見

アクション：友達グループLINEでシェア、放課後一緒に行くことに

価値：日常に小さなサプライズと友達との共有体験

システム側の処理：位置情報とユーザー属性に基づいたスマート通知配信

🏢 9:30 - 行政職員の朝のルーティン（佐藤・区職員）

シチュエーション：出勤前にアプリで市民の声をチェック

発見：渋谷駅周辺で「外国人向けの案内表示がわかりづらい」という投稿が複数

アクション：関連部署にSlackで共有、改善案の検討をスケジュール

価値：市民のリアルな声を迅速に政策に反映

システム側の処理：感情分析とクラスタリングで類似課題を自動グルーピング

🌐 12:00 - 多言語交流の昼休み（ソフィア・外国人）

シチュエーション：ランチを食べながら、近くの投稿を翻訳で読む

発見：「このカフェの店主さん、英語話せるよ！」という日本人の投稿

アクション：そのカフェを訪問、店主と英語で会話、自分も英語で投稿

価値：言語の壁を超えたリアルな地元体験

システム側の処理：DeepL APIによるリアルタイム翻訳、文化的配慮の表示

☕ 14:30 - カフェオーナーの集客（田中・店舗経営）

シチュエーション：午後の客足が落ち着いたタイミングで新メニューを投稿

投稿内容：「秋限定！栗のヴィーガンケーキできました🌰 15:00〜試食会やってます」

結果：30分後に近くにいた3組のお客さんが来店

価値：リアルタイムマーケティングによる効果的な集客

システム側の処理：近隣ユーザーへの位置ベース通知、ビジネス分析データ提供

🎨 16:00 - Quest参加でコミュニティ貢献（李・留学生）

参加：研究の知見も活かしながら、駅の案内表示改善案を写真付きで投稿

反応：他の外国人ユーザーから共感のいいね、佐藤さんからも区役所の公式リプライ

🚀 クイックスタートガイド

1. 環境構築

# Node.js 18.x と npm をインストール
npm install -g @react-native-community/cli

# リポジトリクローン
git clone https://github.com/shibuya-live-canvas/app.git
cd shibuya-live-canvas

# 依存関係インストール
npm install

# iOS の場合
cd ios && pod install && cd ..

2. 環境変数設定

# .env.local ファイル作成
cp .env.example .env.local

# 必要な API キーを設定
DEEPL_API_KEY=your-deepl-api-key
MAPBOX_ACCESS_TOKEN=your-mapbox-token
FIREBASE_CONFIG=your-firebase-config.json

3. 開発サーバー起動

# React Native Metro サーバー起動
npm start

# iOS シミュレーター
npm run ios

# Android エミュレーター
npm run android

# Web 開発環境（管理画面）
npm run web
